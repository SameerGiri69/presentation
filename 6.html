System Workflow

Transfer Process: 4-Stage Overview

01

The Radar

Discovery

Peers broadcast presence to the local network to find each other automatically.

UDP Multicast
02

The Request

Handshake

Sender initiates connection and sends file metadata (name, size, ID).

TCP + JSON
03

The Gatekeeper

Decision

Receiver gets a prompt. Connection is held open until accepted or rejected.

User Interaction
04

The Highway

Streaming

File data flows in chunks directly to the Downloads folder with live progress.

TCP Stream (64KB)

The separation of Discovery (UDP) and Transfer (TCP) ensures that heavy file transfers don't clog the discovery channel.
