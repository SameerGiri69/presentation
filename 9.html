Accept / Reject Logic

Security is paramount. The connection is established but effectively "frozen" until the receiver explicitly authorizes the transfer. This prevents unsolicited files from flooding the disk.

  • Connection Holding

    The TCP socket stays open in a read-wait state. No file data is transferred until an affirmative response is sent.

  • User Prompt (UI Toast)

    Receiver sees a non-blocking toast: "Sameer wants to send 'Video.mp4' (2GB). Accept?"

  • Explicit Confirmation

    Clicking "Accept" triggers the JSON response. "Reject" immediately terminates the TCP connection.
