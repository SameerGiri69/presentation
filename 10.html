Chunked Streaming

Once accepted, the file is not loaded entirely into RAM. Instead, it flows in small, manageable 64KB chunks through the TCP connection, ensuring memory stability even for large files.

  • Stable Memory Usage

    By using a fixed 64KB buffer, the app uses constant memory regardless of whether the file is 1MB or 10GB.

  • TCP Reliability

    The TCP protocol automatically handles packet ordering and retransmission, ensuring the file arrives uncorrupted.

  • Direct-to-Disk

    The receiver writes incoming chunks immediately to the Downloads folder, minimizing system overhead.

transfer_service.go
// 64KB buffer for reading const BufferSize = 64 * 1024 buf := make([]byte, BufferSize) for { n, err := file.Read(buf) if err != nil { break } // Write chunk to TCP socket conn.Write(buf[:n]) // Update DB & Notify UI UpdateProgress(transferID, n) }
