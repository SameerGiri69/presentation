02

TCP Connection

Once a peer is selected, the Sender initiates a direct TCP connection. The first packet sent is not the file content, but metadata describing the transfer request.

  • Metadata First

    Sends a lightweight JSON packet containing filename, size, and sender identity before any heavy data.

  • Dedicated Channel

    Establishes a reliable, ordered TCP socket specifically for this file transfer session.

  • Pending State

    The transfer pauses immediately after the handshake, waiting for the Receiver's explicit approval.
